# 任务上下文梳理与续接

这个功能用于重建“现在真正有效的任务上下文”，而不是按时间顺序压缩整段聊天记录。它适合长对话、多任务交错、方案反复讨论，或需要把工作交给新会话继续的场景。

## 调用方式

只有当前消息显式调用 AgentTools 时才会运行：

```text
@AgentTools 总结当前任务上下文
@AgentTools 生成可供新会话继续的上下文
@AgentTools 只整理当前未完成事项
```

安装 Skill 不会自动读取或总结会话，之前某一轮的调用也不会延续到下一轮。

## 梳理原则

1. 将长对话拆分成不同任务，并标记为已完成、进行中、暂停或状态未知。
2. 以用户最新的明确决定为准；在已有证据中，以实际检查过的代码、文件和工具结果为准。
3. 较早内容只有在仍然约束当前任务、解释必要因果关系，或记录需要避免重走的失败路线时才保留。
4. 不补写不存在的信息。缺失内容、假设和未经验证的说法会被明确标记。
5. 代码任务优先说明“入口 → 路由 → 核心模块或步骤 → 状态与输出”，并保留简要做法和重要约束，不堆叠例行命令日志。
6. 提案、备选方案和用户已经采用的决定会被区分，未决定的方案不会被擅自合并或写成既定事实。

## 未决方案与提问

不会把长对话里出现过的所有分歧都展示给用户。只有仍然可行、会实质影响当前任务或下一步的分支才进入“待确认分支”。

默认不提问：只要可以安全地把分支保留为未决，就直接生成总结。确实必须确认时，只进行一轮合并提问，最多列出 3 个关键决定，并允许用户继续保留未决项。

如果用户只回答一部分或没有回答，剩余项会保持待定，不会再次追问。只有用户明确要求继续解决剩余选择时，才会开启下一轮确认。

## 默认输出

没有指定格式时，按需输出以下部分，空部分会省略：

- 当前任务主线
- 当前代码或执行流程
- 关键做法与约束
- 当前有效状态
- 待确认分支
- 下一步

输出应足以让新会话继续工作，同时清楚区分已验证事实、用户决定、合理假设和未决分支。

## 边界

- 只使用当前可访问的对话与已经获得的证据；无法访问的早期上下文会明确说明。
- 不展示隐藏指令、私密推理、密钥、凭据或无关原始日志。
- 不因“总结上下文”自动扫描无关文件、运行诊断或修改项目。
- 除非用户明确要求生成文件，否则不会把总结写入磁盘或声称它是跨会话记忆。
